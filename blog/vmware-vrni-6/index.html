VMware vRealize Network Insight 文档

发现、优化应用安全性和网络连接解决方案并对其进行故障排除

VMware vRealize Network Insight 可帮助您跨混合和多云环境构建经过优化且高度可用的安全网络基础架构。它提供了网络可见性和分析功能，以提高微分段安全性，最大限度地降低应用迁移期间的风险，优化网络性能 (sysin)，并信心十足地管理和扩展 NSX、SD-WAN Velocloud 和 Kubernetes 部署。

应用场景

网络可见性和分析

  • 混合和多云环境以及 SD-WAN 的可见性，支持 VMware NSX、VeloCloud、VMware Cloud on AWS 和 Microsoft Azure
  • 发现叠加和底层网络之间的连接
  • 分析混合云中的流量和应用

规划应用安全性和云迁移

  • 发现应用（虚拟机、容器），确定流量模式
  • 规划安全性：推荐防火墙策略、微分段应用
  • 跨虚拟机、容器和云环境排除混合应用的安全故障
  • 映射依赖关系以降低应用迁移期间的风险

优化虚拟网络和物理网络并进行故障排除

  • 缩短应用连接问题的平均修复时间 (MTTR)
  • 通过消除网络瓶颈优化应用性能 (sysin)
  • 跨多个 NSX Manager 进行管理和扩展

新增内容

以下是 vRealize Network Insight 6.6 的主要特性和功能：

VMware NSX-T 监控和故障排除

  • vRealize Network Insight 支持在网络映射和意向中对裸机 Edge 使用 VMware NSX Firewall。
  • vRealize Network Insight 引入了 Crown Jewel 分析，以确定在安全违规期间可能发生的横向移动。

网络保证和验证

  • vRealize Network Insight 在网络映射和虚拟机-虚拟机路径中支持 Arista BGP-EVPN。
  • vRealize Network Insight 支持在网络映射中使用 Cisco UCS FEX。

平台增强功能

  • 您可以对 vRealize Network Insight 管理的数据源执行批量操作，以启用、禁用、删除和更新参数，如凭据、轮询间隔和备注。这样可以节省时间，并帮助您一次性管理多个数据源。
  • 可以在部署 vRealize Network Insight 时指定安全网络时间协议 (NTP) 服务器以获取时间戳。安全 NTP 支持网络时间，并可为网络时间协议的客户端-服务器模式提供加密安全性。
  • 您可以从命令行界面使用自动网络发现来批量添加物理设备，例如路由器和交换机。
  • 您可以在同一屏幕上查看东西向和南北向 IP 范围。还可以在编辑 IP 范围以替代用于流量分析的 RFC 1918 规范时添加例外。

应用程序可用性增强功能

  • 添加应用程序时，您现在可以在保存更改之前查看微分段轮形视图。这有助于您更好地了解应用程序流。
  • 现在，您可以在应用程序拓扑图中查看负载均衡器。您也可以选择在负载均衡器六边形和层六边形之间运行的 Edge 以查看完整的流信息。
  • 在执行基于流的应用程序发现时，您可以选择负载均衡器作为应用程序命名首选项。此外，在完整发现的前 7 天之后，基于流的应用程序发现时间将从 6 小时缩短到 3 小时。

数据总线增强功能

  • vRealize Network Insight 引入了检测信号消息，告知订阅者数据总线流式传输路径是否可用。当订阅者看不到来自已订阅消息组的任何传入消息时，此功能对于进行故障排除非常有用。
  • vRealize Network Insight 引入了新的消息组以支持粒度级别的衡量指标数据。实体特定衡量指标的消息组包括：
    • vms-metrics
    • hosts-metrics
    • flows-metrics
    • nics-metrics
    • switchports-metrics

其他

  • vRealize Network Insight 6.6 使用 Log4j 2.17.1。

    对于修复不受 VMware 控制的第三方组件，我们移除了 JndiLookup 类以缓解报告的漏洞。此外，还会设置标记 log4j2.formatMsgNoLookups = true。

    注意：如果使用了 JDBC Appender，则 CVSS3 评分为 6.6 的漏洞 https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-44832 可被利用。vRealize Network Insight 不受此漏洞影响，因为未使用 JDBC Appender。

    要解决以前版本的 Log4j 漏洞，请参见 KB-87135。

  • vRealize Suite Lifecycle Manager 8.7 产品支持包 1 支持安装 vRealize Network Insight 6.6。请参见 VMware vRealize Suite Lifecycle Manager 8.7 PSPAK 1 发行说明。要使用 vRealize Suite Lifecycle Manager 安装和升级 vRealize Network Insight，请参见《vRealize Suite Lifecycle Manager 安装、升级和管理指南》
