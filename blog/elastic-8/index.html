作者：Dan Courcy，George Kobar 2022年2月11日

我们很高兴地宣布 Elastic 8.0 全面上市。凭借对 Elasticsearch 向量搜索功能的增强、对现代自然语言处理模型的原生支持、日益简化的数据载入以及简化的安全体验 (sysin)，Elastic 8.0 开创了一个速度、规模、相关性和简单性的新时代。

无论是 将人员和团队与重要内容联系起来，保持任务关键型应用程序和基础设施在线，还是 保护整个数字生态系统免受日益复杂的网络威胁，Elastic 8.0 都在为下一代搜索驱动的解决方案奠定基础，以帮助每个人接触新的成功水平。

准备好开始了吗？ Elastic 8.0 现已在 Elastic Cloud 上推出，这是唯一一款包含此最新版本中所有新功能的托管 Elasticsearch 产品。

速度、规模和相关性：新的开始，相同的基础

每一个结束都有一个新的开始。而且，随着我们都开始新的一年（见 2021 年，你好 2022 年）——我们也开始进入与 Elastic 8.0 相关的速度、规模和相关性的新时代。

我们的客户和社区都知道，我们对速度、规模和相关性的承诺是坚定不移的。每个 Elastic 版本都有增强和优化，以确保 Elasticsearch 成为最快、最具可扩展性和最强大的搜索引擎。

事实上，在过去的三年里，我们在以下方面取得了长足的进步：减少内存使用（允许每个节点管理更多数据）、减少查询开销（特别是对大型部署有影响），并引入一些全新的特性来增强相关性.

例如，在 7.x 版本流中，我们提高了 日期直方图和 搜索聚合，增强了 页面缓存，并创建了一个新的 “预过滤”搜索阶段。此外，我们通过 减少内存堆、的全面支持 ARM、引入使用 更少存储的新方法以及使我们的客户能够轻松地将计算与存储分离 具有新的 冻结层和可搜索的快照。

也许关于无尽的 Elastic Stack 优化流的最好的部分是，无论您选择如何让数据发挥作用，这些增强功能本质上可以帮助您快速、大规模地搜索、解决问题并取得成功——无需繁琐的工作。

通过原生矢量搜索提高搜索相关性

Elastic 8.0 带来了一整套原生矢量搜索功能，使客户和员工能够使用自己的文字和语言搜索和接收高度相关的结果。

在过去的两年里，我们一直在努力使 Elasticsearch 成为进行矢量搜索的好地方。回到过去，随着 Elasticsearch 7.0，我们为高维向量引入了字段类型。在 Elasticsearch 7.3和 Elasticsearch 7.4中，我们引入了对向量相似度函数的支持。这些早期版本展示了将矢量搜索技术引入 Elasticsearch 生态系统的前景。我们很高兴看到我们的客户和社区急切地将它们用于各种用例。

今天，借助 Elasticsearch 8.0，我们将自然语言处理 (NLP) 模型的原生支持直接引入 Elasticsearch，从而使向量搜索的实施更加实用。此外，Elasticsearch 8.0 包括对 近似最近邻(ANN) 搜索的原生支持——这使得基于向量的查询与基于向量的文档语料库的速度和规模进行比较成为可能。

借助 NLP 的力量打开分析的新世界

Elasticsearch 一直是进行 NLP 的好地方，但从历史上看，它需要在 Elasticsearch 之外进行一些处理，或者编写一些非常复杂的插件。借助 8.0，用户现在可以 在 Elasticsearch 中更直接 等操作——无需额外的组件或编码。不仅在 Elasticsearch 中本地计算和创建向量在水平可扩展性方面是“胜利”（通过在服务器集群中分布计算）——这一变化还为 Elasticsearch 用户节省了大量时间和精力。

快速搜索，大规模搜索

鉴于 Elastic 8.0 基于 Lucene 9.0，那些利用现代 NLP 的搜索体验可以通过（新的）对近似最近邻搜索的原生支持，快速、大规模地实现这一目标 (sysin)。ANN 可以快速有效地将基于向量的查询与基于向量的文档语料库（无论是小型、大型还是巨大的语料库）进行比较。

并认为这仅仅是个开始……

总之，凭借对现代 NLP 模型的原生支持和对 ANN 搜索的原生支持，Elastic 8.0 为我们的客户和社区释放了矢量搜索的力量。要了解更多信息，请加入我们参加 网络研讨会，NLP 模型和矢量搜索 有关使用 PyTorch 进行 NLP 的博客。

未来，搜索驱动的解决方案（如 Elastic Enterprise Search）将使利用矢量搜索的强大功能变得非常简单。在此示例中，用户的搜索词不必与最佳结果完全匹配。矢量搜索能够“连接点”并轻松显示最相关的结果。

简单的事情应该简单

  • 通过两个新的 Amazon Web Services (AWS) 集成简化云原生可观察性

在 过去 几个 版本中，我们简化了从任何来源将任何类型的数据获取到 Elastic Stack 的过程。在 Elastic 8.0 中，我们通过两个新的 AWS 集成进一步扩展了我们的云原生集成，即：一个新的 AWS Lambda 应用程序和一个新的 Amazon Simple Storage Service (Amazon S3) Storage Lens 集成。

的 新 AWS Lambda 应用程序在 AWS Serverless Application Repository (SAR) 中发布 简而言之，用户现在只需在 AWS 控制台中单击几下，即可将来自 Amazon S3 的日志提取到他们的 Elastic Cloud 部署中。

的 Amazon S3 Storage Lens 集成，于 12 月首次宣布用户可以轻松地将 Storage Lens 指标传送到他们的 Elastic Cloud 部署中，使用 Elasticsearch 强大的搜索和分析功能以及 Kibana 强大的仪表板功能来优化 Amazon S3 使用成本，确保数据保护，并监控用户活动趋势。

  • 通过简化的堆栈安全性保护您的数据免受未经授权的访问

适当的安全性从未像今天这样重要。尽管安全功能多年来一直 免费并包含在 Elastic Stack 中，但我们已经简化了配置安全性所需的步骤——因为我们认为任何集群都不应在不受保护的情况下运行。在 Elastic 8.0 中，现在默认为自管理集群启用安全性。这可确保数据、网络和用户在 Elastic Stack 中得到保护，防止数据泄露和未经授权的访问。借助自动生成的令牌和证书等产品内帮助，我们简化并简化了此过程，以节省时间和精力，让所有人都能获得安全性。

而且，如果您已经在使用（或准备尝试）Elastic Cloud，请放心 — 始终启用安全性和完全基于角色的访问控制。要了解有关简化堆栈安全性的更多信息，请参阅 详细的技术博客。
