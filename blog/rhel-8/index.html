2021 年 11 月 10 日，红帽企业 Linux (RHEL) 8.5 现已正式发布 (GA)，并带来了新功能和改进，以帮助简化部署、优化性能并帮助降低环境中的风险。无论您是在本地部署 RHEL，亦或在共有云上部署 RHEL，还是在边缘部署 RHEL，以上所有部署方式 (sysin)，RHEL 8.5 中都有一些改进，用户都会渴望深入研究。

RHEL 8.5 的新增功能

RHEL 8.5 中的 Linux 容器改进

RHEL 8.5 延续了运行 Linux 容器的新功能和改进的传统。此版本带来的工具将增加灵活性并减少在更广泛的环境中运行 Podman 的摩擦。

  • 容器化 Podman - RHEL 8 Podman 容器映像 (rhel8/podman) 现已正式发布，可以帮助解锁 Podman 在云 CI/CD 系统、Windows 上的 WSL2、macOS 上的 Docker 桌面下以及（当然）在 RHEL 上的使用 6、7 和 8。您可以使用 Podman 容器镜像来帮助开发和运行其他容器镜像。
  • 默认情况下验证容器镜像签名 - 在 RHEL 8.5 中，用户可以放心地拉取容器镜像。RHEL 8.5 开箱即用，将检查容器映像签名以验证它们实际上来自 Red Hat 并且未被篡改或操纵。
  • 作为无根容器用户的本机 OverlayFS - RHEL 8.5 在构建和运行无根容器时提供更好的性能，并提供对 OverlayFS 的本机支持。

RHEL 8.5：更易于管理和部署

我们将继续专注于简化和改进管理和部署 RHEL 的方法。在 RHEL 8.5 中，我们引入了许多自动化和管理工具来自动执行手动任务、标准化大规模部署并简化其系统的日常管理。

  • 增强的 Web 控制台性能指标 - 有助于识别性能问题。无论您是要确定 CPU、磁盘还是网络性能问题，RHEL 8.5 Web 控制台中提供的增强指标都可以帮助您解决问题。此外，您将能够更轻松地将指标导出到 Grafana 服务器。
  • 用于硬件管理的 Ansible 模块 - 使用 RHEL 8.5，您可以使用 Ansible 来管理智能平台管理接口 (IPMI) 的设置，例如系统的电源状态和设备的引导顺序。
  • VPN 和 Postfix 的系统角色 - 您可以使用系统角色减少设置 VPN 和 Postfix 所需的时间。

管理系统的一部分是处理安全性和策略合规性。为此，RHEL 8.5 具有许多功能 (sysin)，可在部署新系统或管理现有基础架构时帮助管理安全性和合规性。

  • 所有 Red Hat Enterprise Linux 次要版本上的 内核实时补丁 - 现在，内核实时补丁也可用于 RHEL 次要版本，并且能够在特定内核发布后的六个月内访问内核实时补丁。这意味着客户可以更灵活地通过应用关键和重要的安全补丁而无需重新启动来维护其 RHEL 系统的安全性。
  • 通过 Web 控制台进行内核实时修补管理 - 您可以从 RHEL Web 控制台内管理内核实时修补，以降低执行关键维护的复杂性。这项新功能提供了一个简化的界面，无需重新启动即可应用内核更新。
  • 增强的系统安全服务守护进程 (SSSD) 日志记录 - 访问增强的 SSSD 用户身份验证日志，包括完成时间、错误、身份验证流程和配置等详细信息。此增强功能包括搜索功能，因此管理员可以更轻松地分析用户身份验证基础架构中的性能或配置问题。
  • 网络时间协议 (NTP) 的网络时间安全 (NTS) - 应用 NTS 标准，该标准提供 NTP 服务器的加密身份验证。这种身份验证有助于防止可能危及关键计时服务的中间人攻击。

SQL Server 增强功能

在 RHEL 上运行 Microsoft SQL Server 的客户将看到许多增强功能，以帮助更有效地配置、管理和操作 RHEL。

  • Microsoft SQL Server 的 RHEL 系统角色 - 可用于 RHEL 8.5。此系统角色允许 IT 管理员和 DBA 以自动化方式更快地安装、配置和调整 SQL Server。
  • 用于 Red Hat Insights 的 SQL Server 评估 API - 通过提供来自 Microsoft 的 SQL Server 评估 API 的信息，帮助为系统和数据库管理员提供最佳用户体验。它有助于提供来自 Microsoft 的最佳实践来评估 SQL Server 的配置，并为用户提供针对通过 API 发现的问题的修复功能。

对于开发人员和应用程序支持

归根结底，RHEL 就是要运行您的应用程序和基础架构。作为构建和运行下一代应用程序的坚实基础，RHEL 8.5 支持 OpenJDK 17 和 .NET 6。

OpenJDK 17 允许开发人员利用新的 Java 功能，同时还提高了现有应用程序环境的稳定性。OpenJDK 17 中值得注意的功能包括增强的伪随机数生成器，它是许多加密操作的通用基础，代数数据类型简化了复杂数据的建模方式，以及强制执行严格的浮点一致性以更好地预测数字敏感应用程序。

OpenJDK 17 也可用作 RHEL 通用基础映像 (UBI) 容器，为开发人员团队提供了尝试和升级应用程序以及在 CI/CD 管道中使用 OpenJDK 的新方法。

.NET 6 的核心是从 .NET 5 开始的 .NET 统一计划的最后部分。

.NET 6 是具有多项性能改进的长期支持版本。.NET 6 中的新功能是 C# 支持以及改进的诊断和几个新的 API，包括 WebSocket 压缩和简化的加密。
