Announcing SQL Server 2022 preview: Azure-enabled with continued performance and security innovation

2021 年 11 月 2 日

​

​作者：彼得卡林

CVP，Azure 数据库服务

​SQL Server 2022，公告

今天，我们宣布发布 SQL Server 2022 预览版，这是迄今为止支持 Azure 最多的 SQL Server 版本，在性能、安全性和可用性方面不断创新。

数据的兴起代表着巨大的机遇，也带来了挑战。公司看到他们的关系和非关系数据在本地、云、边缘和混合环境中呈指数级增长。最具变革性的公司推动对当前数据的预测性洞察，而其他公司可能难以推动对其历史数据的反应性洞察。信息可能会跨地域和部门孤立。

为了在这种环境中为客户提供支持，Microsoft 提供了一个端到端的产品和服务数据平台，这些产品和服务共同应对这些挑战。操作数据库涵盖所有可能的部署位置，包括 SQL Server 和启用 Azure Arc 的数据服务、Azure SQL 完全托管的云数据库以及用于 IoT 设备的 Azure SQL Edge。为了实现实时洞察，Azure Synapse Analytics 将数据集成、企业数据仓库和大数据分析结合在一起 (sysin)，客户可以使用 Power BI 可视化他们的数据。客户可以使用 Azure Purview 发现、编目和管理其数据所在的任何位置。

SQL Server 2022 与 Azure Synapse Link 和 Azure Purview 集成，使客户能够从其大规模数据中获得更深入的洞察、预测和治理。云集成通过到 Azure SQL 托管实例的灾难恢复 (DR) 以及到云分析的无 ETL（提取、转换和加载）连接得到增强，这使数据库管理员能够以更大的灵活性和最小的影响来管理他们的数据资产最终用户。性能和可扩展性通过内置的查询智能自动增强。跨语言和平台（包括 Linux、Windows 和 Kubernetes）有多种选择和灵活性。

启用 Azure

双向 HA/DR 到 Azure SQL

为确保正常运行时间，SQL Server 2022 与 Azure SQL 托管实例中的新链接功能完全集成。借助 Azure SQL 托管实例的新链接功能，您现在可以获得运行适用于灾难恢复的 PaaS 环境的所有好处——即使与 IaaS 环境相比，您也可以在设置和管理上花费更少的时间。这是通过使用内置的分布式可用性组 (DAG) 将数据复制到以前部署的 Azure SQL 托管实例作为 DR 副本站点来实现的。该实例已准备就绪，可在您需要时随时等待 — 无需冗长的配置或维护。您还可以在读取横向扩展方案中使用此链接功能来卸载可能会影响数据库性能的繁重请求。我们正在努力构建更多功能来支持双向数据移动。

Azure 突触链接

以前，将数据从本地数据库（如 SQL Server）移动到 Synapse 需要您使用 ETL。众所周知，设置和运行 ETL 管道需要大量工作，而且洞察力落后于任何时刻发生的事情。适用于 SQL Server 2022 的 Azure Synapse Link 提供自动更改源，可捕获 SQL Server 中的更改并将这些更改提供给 Azure Synapse Analytics。它提供近乎实时的分析以及混合事务和分析处理，对操作系统的影响最小。数据到达 Synapse 后，您可以将其与许多不同的数据源结合起来 (sysin)，无论它们的大小、规模或格式如何，并使用您选择的 Azure 机器学习、Spark 或 Power BI 对所有数据运行强大的分析。由于自动更改源只会推送新的或不同的内容，因此数据传输速度更快，现在可以实现近乎实时的洞察，同时对 SQL Server 2022 中源数据库的性能影响最小。

Azure 权限集成

我们最近宣布正式推出 Azure Purview 作为统一的数据治理和管理服务。我们很高兴地强调，SQL Server 还与 Azure Purview 集成以实现更好的数据发现，从而打破数据孤岛。通过此集成，您将能够：

  • 免费自动扫描您的本地 SQL Server 以捕获元数据。
  • 使用内置和自定义分类器以及 Microsoft 信息保护敏感度标签对数据进行分类。
  • 设置和控制对 SQL Server 的特定访问权限。

性能、安全性和可用性的增强

表现

SQL Server 提供差异化的性能，OLTP 性能1️⃣在 1TB 上的 和非集群 DW 性能排名第一2️⃣，、3TB3️⃣、10TB4️⃣ 和 30TB5️⃣ 根据独立的事务处理性能委员会。SQL Server 2022 创新中的内置查询智能包括：

  • 对于查询存储，我们正在添加对只读副本的支持并启用查询提示以提高性能并快速缓解问题，而无需更改源 T-SQL。
  • 对于智能查询处理，我们正在根据常见的客户问题扩展更多场景。例如，“参数敏感计划” 问题指的是这样一种情况，即参数化查询的单个缓存计划对于所有可能的传入参数值并不是最佳的。借助 SQL Server 2022 的参数敏感计划优化功能，我们可以为单个参数化语句自动启用多个活动缓存计划的生成。这些缓存的执行计划将根据提供的运行时参数值容纳不同的数据大小。

安全

在过去十年中，SQL Server 的漏洞比竞争对手少。6️⃣ 在此基础上，新的分类账功能创建了一个不可变的数据修改跟踪记录。这可以防止数据被恶意行为者篡改，并且有利于内部和外部审计等场景。

可用性

随着员工和客户向更全球化分布的转变，许多组织正在转向多写入环境，该环境允许对本地数据库进行更改并以双向更新流推送到其他副本。但是，如果多人更改了数据库中的同一行，并且不同的写副本中包含不同的信息，那么以前这种对等副本冲突将导致整个操作停滞，直到它被解决。使用 SQL Server 2022，我们正在自动执行最后写入者获胜规则。现在，当检测到冲突时，将选择最近的修改时间在所有副本上持久化。这有助于保持您的多写场景平稳运行。
